# Technology And Skills

Progression in Valley Claim should come mostly from skills, relationships, local knowledge, and
capacity. Technology matters, but it should spread through the world rather than unlock from a
universal tree.

## Principle

The question is not "has the player researched X?" It is:

- Does this scenario date/place make X possible?
- Is X available through markets, institutions, neighbors, migration, or contact?
- Can the agent afford, maintain, transport, and use X?
- Does the household have the skill and relationships to benefit from X?

## Skill Model

Skills represent practiced ability and social knowledge.

| Skill Area | Affects |
|---|---|
| Farming | field creation, crop yield, drought/frost handling, soil mistakes |
| Irrigation/water | water access, field resilience, illness risk, labor efficiency |
| Building/carpentry | shelter quality, storage, repairs, fences, barns |
| Hunting/trapping | food reliability, risk, tool use, knowledge of terrain |
| Animal handling | transport, traction, food, breeding, loss risk |
| Medicine/care | illness outcomes, recovery, childbirth/injury risk |
| Trade/accounting | prices, debt terms, inventory planning, credit |
| Travel/navigation | route risk, information, distance costs |
| Language/mediation | trust, diplomacy, conflict avoidance, information |
| Law/claims | claim status, disputes, fees, deadlines, appeals |
| Repair/tool use | tool lifespan, work speed, maintenance cost |

Skills should affect concrete outcomes:

- action duration or labor cost
- yield
- accident/failure risk
- quality of result
- available action choices
- information quality
- prices or access
- reputation gain/loss

## How Skills Improve

Use grounded routes:

- repeated work
- mentorship/apprenticeship
- travel and observation
- contact with neighbors or institutions
- books/manuals where plausible
- crisis experience
- deliberate practice at opportunity cost

Avoid generic XP for unrelated actions. Clearing brush should not improve diplomacy; mediating a
dispute should not improve carpentry.

## Technology Diffusion

Technology availability is scenario data plus dynamic spread.

Sources:

- store/trader inventory
- migration of skilled people
- military, mission, church, or government institutions
- rail, road, river, or trail access
- neighboring agents adopting tools
- mail/newspapers/catalogs where time/place supports it
- indigenous, Spanish, Mexican, Mormon, American, or other local knowledge systems depending on
  scenario

Constraints:

- purchase cost
- transport cost
- maintenance and repair
- consumable inputs
- training/skill
- social or legal access
- debt or obligation
- supply disruption

## Technology Categories

| Category | Examples | Gameplay Role |
|---|---|---|
| Hand tools | axe, shovel, hoe, saw, auger | Enable and speed daily work |
| Water systems | ditch, well, pump, cistern | Reduce water labor and crop risk |
| Building methods | dugout, log cabin, adobe, frame building | Shelter quality, materials, maintenance |
| Agriculture | crop varieties, plows, fencing, storage | Yield, resilience, labor tradeoffs |
| Transport | pack animal, wagon, road, rail access | Market reach, hauling, migration |
| Communication | mail, newspaper, telegraph access | Information, prices, events, politics |
| Weapons/security | firearm types, militia access, fort presence | Risk, deterrence, politics |
| Medicine | remedies, trained care, sanitation knowledge | Illness/injury outcomes |

## Player-Facing Rules

When a new technology appears, explain why:

- "The railhead reached the county."
- "A trader has a pump in stock."
- "A neighbor can teach ditching if relations are good."
- "A mission workshop can repair tools."
- "A catalog order is possible, but cash and delivery time are required."

When a technology is unavailable, explain that too:

- missing market access
- wrong period/place
- no skilled person nearby
- too expensive
- missing material
- legal/social barrier

## Avoid

- A single linear tech tree.
- Technologies that are pure buffs with no cost.
- Generic "frontier tech" available in every scenario.
- Treating indigenous, Spanish/Mexican, Mormon, military, or American systems as one undifferentiated source.

## Near-Term Implementation

Start small:

1. Add skill tags to actions.
2. Add a few household skill values.
3. Let skill adjust work results and risk.
4. Add tool availability to the store/trader model.
5. Move tool availability into scenario data.
6. Add one or two technology-spread events tied to date/place.
